3  —  Network & Ports
8501/tcp Streamlit UI Localhost only
11434/tcp Ollama API Localhost only

No inbound internet required.
No outbound internet required after installation.
Firewall rule: block all ports except 8501 on LAN.
Embedding model (all-MiniLM-L6-v2) is cached locally after first download.

4  —  LLM Models (installed via Ollama)
llama3.1:8b ~4.7 GB ollama pull llama3.1:8b Baseline — CPU-friendly, good for Basic and Advanced modes
phi4:14b ~8.9 GB ollama pull phi4:14b Recommended — strongest accuracy, benefits from GPU or 32 GB RAM

Models are downloaded once and stored under ~/.ollama/models/. The active model is configurable per-attorney from within the Amicus UI (Settings → Model).

5  —  Data Storage Layout
./db/chroma/ ChromaDB vector store — grows with ingested documents (~5–50 MB per case)
./db/embeddings/ Cached sentence-transformer model (~90 MB, downloaded once)
./db/user_config.json Active LLM model preference, per-machine setting
./feedback/ Anonymized query feedback logs (local only, never transmitted)

Case documents are processed at upload time; original files are not stored by Amicus. Only extracted text chunks and their embeddings are retained in ChromaDB.

6  —  Installation Reference
macOS / Linux bash install_amicus.sh
Launch (macOS) ./launch.sh   # or double-click launch.command
Windows (run as Administrator) terminal_setup.bat
Launch (Windows) launch.bat

See QUICK_START.md for step-by-step instructions including GPU driver setup and model downloads. Run python reingest.py to rebuild the vector store if the ChromaDB index becomes corrupted.

Security posture: Amicus does not make outbound connections during normal operation. All LLM inference runs locally via Ollama. Case documents, queries, and responses never leave the machine. No cloud accounts, API keys, or external licensing servers are required after initial installation.
