Network Setup Options

Option A — Dedicated private router simplest

A standalone router connected only to the Mac Mini. Attorneys connect their laptops to this network when using Amicus. Fully air-gapped — router has no WAN connection. Setup time: ~30 minutes.

Option B — Firm LAN with firewall isolation

Mac Mini lives on the existing firm network, firewall-isolated from the internet. More convenient for attorneys — no network switching. Requires IT configuration. Amicus traffic never leaves the LAN segment.

Attorney access: Each attorney opens a browser and navigates to http://[server-ip]:8501 — no software installation required on laptops. Works on Mac, Windows, iPad. The Amicus interface is identical to the standalone version.   ✕ no internet  No case data, no queries, no documents ever leave the private network.

What Runs on the Server

AI reasoning engine — Llama 3.3 8B via Ollama. Processes all queries locally. No API keys, no cloud inference costs, no per-query fees.

Document store — ChromaDB vector database. Stores semantic embeddings of all case documents. Supports semantic search, keyword search, and reranking.

Amicus application — Streamlit web server. Handles all attorney browser sessions simultaneously. Routes queries, manages cases, streams responses.

File storage — Original documents stored on the SSD alongside their processed embeddings. Persists indefinitely.

PII redaction pipeline — Runs at ingestion time. Strips names, addresses, and identifiers from documents before they enter the searchable corpus.

What runs on attorney laptops — A browser. Nothing else. All AI, retrieval, and storage is server-side.
